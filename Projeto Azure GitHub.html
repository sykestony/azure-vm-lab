# Laboratório Prático: Gerenciamento de Máquinas Virtuais no Microsoft Azure

## 📚 Descrição

Este repositório foi criado como parte do desafio da DIO sobre o gerenciamento de máquinas virtuais no Azure. Aqui você encontrará resumos, anotações, dicas e imagens relacionadas às práticas realizadas.

## 🎯 Objetivos

- Aplicar os conceitos aprendidos sobre Azure.
- Documentar o processo técnico de forma clara.
- Utilizar o GitHub para compartilhamento de documentação técnica.

## ✅ O que foi feito

- Criação e configuração de máquinas virtuais (VMs) no Microsoft Azure.
- Gerenciamento de recursos: criação, inicialização, parada e exclusão.
- Configuração de regras de firewall e grupos de segurança.
- Conexão remota via RDP e SSH.
- Boas práticas de segurança no gerenciamento de VMs.

## 📝 Resumo das etapas

### 1. Criação da Máquina Virtual
- Acesse o portal do Azure.
- Vá em "Máquinas Virtuais" → "Adicionar".
- Escolha:
  - Imagem: Ubuntu Server ou Windows Server.
  - Tamanho: baseado na carga de trabalho.
  - Autenticação: senha ou chave SSH.
- Revise e crie.

### 2. Configuração de Rede
- Criação de um grupo de segurança.
- Abertura de portas (por exemplo, porta 22 para SSH, 3389 para RDP).
- Configuração de IP público.

### 3. Conexão com a VM
- **Linux**: via terminal com `ssh user@ip`.
- **Windows**: via Remote Desktop.

### 4. Gerenciamento
- Iniciar, parar e excluir VMs.
- Monitoramento com o Azure Monitor.
- Backup e snapshots.

### 5. Boas Práticas
- Utilizar autenticação por chave SSH.
- Criar regras restritivas de firewall.
- Automatizar tarefas com Azure CLI.

## 💡 Dicas importantes

- Use **tags** nos recursos para organização.
- Configure **alertas** de uso para evitar cobranças excessivas.
- Explore o **Azure Bastion** para conexões seguras.
